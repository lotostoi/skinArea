# Роадмап MVP SkinsArena

## Принцип: вертикальные срезы

Каждая неделя = **готовый кусок продукта**, который можно открыть в браузере и показать. Бэкенд и фронтенд одной фичи делаются вместе. Внешние интеграции (Steam Trade, платёжка) — в конце, когда весь UI и внутренняя логика уже работают.

## Что уже СДЕЛАНО (Неделя 0 — фундамент)

- **БД:** 15 миграций — users, balances, transactions, market_items, deals, cases/levels/items, case_openings, upgrades, withdrawal_requests
- **Модели:** 12 моделей с relations, scopes, casts; 10 PHP Enums
- **Авторизация (бэкенд):** Steam OpenID через Socialite, Sanctum token (код + exchange), logout
- **Админка Filament:** тёмная тема, панель `/admin`: пользователи, балансы, транзакции, заявки на вывод, кейсы (категории/уровни/предметы), маркет
- **API (чтение):** маркет листинг, кейсы, сделки, балансы, транзакции, профиль
- **Form Requests + API Resources** для всех моделей
- **Docker:** PHP 8.4, PostgreSQL 16, Redis, Nginx, Queue, Scheduler

## Неделя 1. Авторизация и каркас сайта

> **Что показываем:** открываешь сайт → тёмная тема в стиле ggdrop → кнопка «Войти через Steam» → логинишься → видишь свой аватар, ник и баланс в хедере. Есть навигация: Маркет, Кейсы, Апгрейд, Профиль.

### Бэкенд

- BalanceService (`app/Services/BalanceService.php`) — charge, credit, hold, releaseHold, transferHoldToSeller. Каждый метод: `DB::transaction` + `lockForUpdate` + запись Transaction
- `app/Exceptions/InsufficientBalanceException.php`
- `app/Exceptions/InvalidDealStateException.php`
- SPA fallback route в `web.php` (все не-API пути → `index.blade.php`)
- Blade entry point для Vue SPA

### Фронтенд

- Установить Vue 3 + TypeScript + Pinia + Vue Router + @vueuse/core
- `resources/js/app.ts` — точка входа
- `resources/js/router/index.ts` — маршруты: `/`, `/market`, `/cases`, `/cases/:id`, `/upgrade`, `/profile/`*
- `resources/js/stores/auth.ts` — Pinia store: user, token, isAuthenticated, login/logout
- `resources/js/stores/balance.ts` — баланс пользователя, автозагрузка после логина
- `resources/js/utils/api.ts` — axios с Bearer interceptor и 401 → logout
- `resources/js/types/` — TypeScript типы: User, Balance, MarketItem, Deal, GameCase и т.д.
- `resources/js/layouts/MainLayout.vue` — тёмный фон `#0e0e12`, хедер `#12121a` (логотип «SkinsArena», навигация, баланс, аватар/кнопка Steam), контент по центру ~1400px
- `resources/js/pages/AuthSteamCompletePage.vue` — обработка `?code=` после callback, обмен на токен
- `resources/js/pages/ProfilePage.vue` — аватар, SteamID, trade URL (редактирование), баланс
- `resources/js/components/ui/` — Button, Modal, Input, Badge, Card, Spinner
- Tailwind config с палитрой из дизайн-системы (`documentation/06-design-system.md`)

### Критерий готовности

Пользователь заходит на сайт → видит тёмную тему → нажимает «Войти через Steam» → проходит авторизацию → возвращается на сайт → видит свой ник, аватар и баланс «0.00 ₽» в хедере → может перейти по навигации.

## 3 вещи, которые нельзя делать плохо даже в MVP

1. **Финансовые операции** — каждая операция с деньгами через `DB::transaction()` + `lockForUpdate()`. Никаких «потом обернём».
2. **Цикл P2P-сделки** — чёткий state machine с атомарными переходами. Два покупателя не могут купить один предмет.
3. **Экономика кейсов** — шансы настраиваются в админке, генерация только на сервере, клиент показывает анимацию.
