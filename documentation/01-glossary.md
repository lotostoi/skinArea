# Глоссарий терминов проекта SkinsArena

## О чем этот документ

Здесь собраны все термины из технического задания SkinsArena, которые могут быть непонятны.
Термины расположены от самых сложных к простым. Каждый объясняется сначала простым языком на примере из жизни, а потом дается техническая суть.

## Сложные термины (архитектура и системы)

---

### 1. Атомарные операции

**Простым языком:**
Представь, что ты переводишь деньги другу. Деньги должны одновременно уйти с твоего счета и прийти на его. Если что-то пошло не так посередине (например, свет выключили), деньги не должны зависнуть "в воздухе" — они либо переведены полностью, либо не переведены вовсе. Это и есть атомарность — операция выполняется целиком или не выполняется совсем.

**Зачем в проекте:**
Когда покупатель покупает скин — деньги списываются с его баланса, переводятся продавцу, предмет меняет статус. Если на любом шаге случится сбой, все должно откатиться назад. Иначе деньги спишутся, а предмет не передастся.

**Техническая суть:**
Атомарная операция — это набор действий, которые выполняются в рамках одной "транзакции" базы данных. Если любое действие внутри транзакции провалилось, все остальные отменяются (rollback). В нашем проекте это критично для: покупки на маркете, открытия кейсов (списание баланса + выдача приза), апгрейда.

### 2. WebSocket

**Простым языком:**
Обычный сайт работает как почта: ты отправляешь запрос — сервер отвечает. Если хочешь узнать новости, надо снова отправить запрос. WebSocket — это как телефонный звонок: связь постоянная, сервер может сам в любой момент "сказать" тебе что-то новое без твоего запроса.

**Зачем в проекте:**

- Live-лента кейсов (предметы появляются в реальном времени у всех)
- Уведомления продавцу "Ваш предмет купили!" (мгновенно, со звуком)
- Обновление статусов сделок без перезагрузки страницы

**Техническая суть:**
WebSocket — протокол двусторонней связи поверх TCP. Клиент (браузер) устанавливает постоянное соединение с сервером. Через него сервер может "пушить" данные клиенту. В отличие от обычных HTTP-запросов, где клиент всегда инициирует общение, тут сервер сам присылает обновления. Реализуется через Socket.io (Node.js) или Laravel Echo + Pusher/Soketi (Laravel).

### 3. Браузерное расширение (Chrome Extension)

**Простым языком:**
Это маленькая программа, которая встраивается прямо в твой браузер (Chrome, Firefox и т.д.). Ты ее устанавливаешь из магазина расширений (как приложение из App Store, только для браузера). После установки у тебя появляется иконка рядом с адресной строкой, и расширение может делать что-то дополнительное на сайтах.

**Зачем в проекте:**
В ТЗ написано "приложение под браузер" — имеется в виду именно расширение. Оно нужно только для одной функции: выставлять предметы на продажу. Без расширения можно покупать, смотреть инвентарь, выводить — но продавать нельзя. Расширение связывается с сайтом через специальный токен (ключ), который пользователь генерирует в личном кабинете.

**Техническая суть:**
Browser Extension — мини-приложение для браузера, написанное на JavaScript/HTML/CSS. Живет в песочнице браузера, имеет доступ к защищенному хранилищу (chrome.storage). Общается с API сайта через HTTP-запросы, авторизуется по API-токену. Нужно публиковать в Chrome Web Store, Firefox Add-ons, Edge Add-ons — каждый магазин имеет процесс модерации (от нескольких дней до нескольких недель).

### 4. JWT-токены

**Простым языком:**
Когда ты заходишь на сайт и вводишь пароль, сайт выдает тебе "пропуск" — зашифрованную строку текста. При каждом следующем действии ты показываешь этот пропуск, и сайт понимает кто ты, без повторного ввода пароля. Этот пропуск и есть JWT-токен.

**Зачем в проекте:**

- Авторизация API (каждый запрос к серверу содержит токен)
- Браузерное расширение авторизуется через отдельный API-токен
- Админ-панель и панель модераторов используют свои токены

**Техническая суть:**
JWT (JSON Web Token) — стандарт токенов авторизации. Содержит три части: заголовок, полезная нагрузка (данные о пользователе, роль, срок действия) и подпись (гарантирует, что данные не подделаны). Передается в заголовке Authorization каждого HTTP-запроса. Не хранится на сервере — сервер просто проверяет подпись.

### 5. Steam OpenID

**Простым языком:**
Вместо того чтобы придумывать логин и пароль на нашем сайте, пользователь нажимает "Войти через Steam". Его перебрасывает на сайт Steam, где он подтверждает вход. Steam говорит нашему сайту: "Да, это настоящий пользователь, вот его Steam ID". Сайт создает аккаунт автоматически.

**Зачем в проекте:**
Это единственный способ авторизации на сайте (по ТЗ). Нет регистрации через email/пароль — только через Steam. Это стандарт для всех сайтов по торговле скинами, потому что нужна привязка к Steam-аккаунту для торговли.

**Техническая суть:**
Steam OpenID — протокол авторизации через Steam. Работает по принципу OAuth: сайт перенаправляет пользователя на Steam → Steam подтверждает личность → Steam перенаправляет обратно с данными о пользователе (Steam ID, аватар, ник). Важно: Steam OpenID отдает только базовую информацию. Дополнительные данные (инвентарь, трейд-бан) запрашиваются отдельно через Steam Web API.

### 6. P2P-маркетплейс

**Простым языком:**
"P2P" значит "от человека к человеку" (peer-to-peer). Это как Авито — продавец выставляет товар, покупатель покупает напрямую у продавца. Сайт не является посредником, он только сводит людей и берет комиссию. Скин передается напрямую от продавца к покупателю через Steam.

**Зачем в проекте:**
Это основная модель торговли на сайте. Сайт не хранит скины у себя — они всегда в Steam-инвентаре пользователей. При покупке создается Steam Trade Offer (предложение обмена) от продавца к покупателю. Деньги при этом проходят через баланс на сайте.

**Отличие от обычного маркетплейса:**
На обычном маркетплейсе (типа магазина) товар сначала отправляется на склад площадки, а потом со склада покупателю. В P2P товар идет напрямую. Это быстрее, но сложнее — нужно следить за тем, чтобы продавец действительно передал предмет.

### 7. Trade Offer / Steam Trade API

**Простым языком:**
Trade Offer — это предложение обмена в Steam. Один игрок предлагает другому: "Я даю тебе свой скин, а ты мне — ничего (или другой скин)". Второй игрок может принять или отклонить. Steam Trade API — это способ создавать такие предложения программно, без ручных действий в Steam.

**Зачем в проекте:**
Когда покупатель купил скин на сайте — сайт автоматически создает Trade Offer от продавца к покупателю. Продавец нажимает "Передать" на сайте, и система через API создает предложение обмена в Steam. Покупатель принимает его в Steam.

**Техническая суть:**
Steam Web API предоставляет методы для работы с трейдами: создание, отслеживание статуса, отмена. Для создания трейда нужна Trade URL (ссылка на обмен) получателя. API также позволяет проверять инвентарь пользователя, наличие Trade Ban и статус Steam Guard.

### 8. Нераспределенный призовой фонд (экономика кейсов)

**Простым языком:**
Представь копилку. Каждый раз, когда кто-то открывает кейс за 500 рублей и получает приз за 250 рублей, разница (250 рублей) падает в копилку. Когда в копилке накопилось достаточно, система "разрешает" выпадение дорогих призов. Если копилка пустая — выпадают только дешевые (гарантированные) призы.

**Зачем в проекте:**
Это защита экономики. Без этой системы сайт мог бы раздать дорогих скинов на миллион, а заработать только тысячу. Фонд гарантирует, что дорогие призы выпадают только когда сайт уже заработал достаточно, чтобы "покрыть" их стоимость (причем с запасом — фонд должен быть в 2 раза больше приза).

**Техническая суть:**
Для каждой категории кейсов ведется отдельный баланс: `Потрачено - Выдано = Нераспределенный фонд`. Приз уровня N может выпасть, только если `Фонд >= 2 * Стоимость_приза`. Ежедневно фонд уменьшается на 5% (настраиваемо) — это доход проекта. Все операции с фондом должны быть атомарными.

### 9. 2FA (двухфакторная аутентификация)

**Простым языком:**
Это двойная проверка при входе. Сначала вводишь пароль (первый фактор), потом код из приложения на телефоне или SMS (второй фактор). Даже если кто-то узнает пароль, без телефона войти не сможет.

**Зачем в проекте:**
2FA требуется для входа в админ-панель (п.16 ТЗ). Это защита от взлома — если злоумышленник получит пароль администратора, он все равно не войдет без второго фактора.

**Техническая суть:**
Обычно реализуется через TOTP (Time-based One-Time Password) — приложения типа Google Authenticator генерируют 6-значный код, который меняется каждые 30 секунд. Сервер и приложение знают общий секрет, по которому синхронно генерируют одинаковые коды.

## Средние термины (предметная область)

---

### 10. Hold Balance (удержанный баланс)

**Простым языком:**
Когда покупатель оплатил скин, деньги не сразу попадают продавцу. Они "замораживаются" — их видно, но потратить нельзя. Это как оплата через Авито Доставку: деньги списались у покупателя, но продавец получит их только после того, как покупатель подтвердит получение. Если что-то пошло не так — деньги вернутся покупателю.

**Зачем в проекте:**
Защита от мошенничества. Деньги на удержании до полного завершения Steam-трейда. Если продавец не передал предмет или отменил трейд — деньги возвращаются покупателю.

---

### 11. Steam Protection (7-дневная защита трейдов)

**Простым языком:**
Steam по умолчанию "держит" переданные предметы 7 дней. Если в этот период трейд отменяется, предмет возвращается. Это встроенная защита Steam от кражи аккаунтов — если аккаунт украли и начали раздавать скины, владелец может отменить трейды в течение недели.

**Зачем в проекте:**
Система должна учитывать эту задержку. Деньги продавца остаются на удержании до окончания 7-дневного периода. Только после этого сделка считается полностью завершенной, и деньги переходят на основной баланс.

---

### 12. Float Value / Wear (износ скина)

**Простым языком:**
У каждого скина есть "степень потертости" — число от 0 до 1. Чем ближе к нулю, тем лучше скин выглядит (новее). Это как состояние б/у товара: может быть "как новый", а может быть "сильно поношенный".

**Категории износа:**


| Сокращение | Полное название                        | Диапазон Float | Простое описание    |
| ---------- | -------------------------------------- | -------------- | ------------------- |
| FN         | Factory New (Прямо с завода)           | 0.00 – 0.07    | Идеальное состояние |
| MW         | Minimal Wear (Немного поношенное)      | 0.07 – 0.15    | Почти новое         |
| FT         | Field-Tested (После полевых испытаний) | 0.15 – 0.38    | Среднее состояние   |
| WW         | Well-Worn (Поношенное)                 | 0.38 – 0.45    | Заметный износ      |
| BS         | Battle-Scarred (Закаленное в боях)     | 0.45 – 1.00    | Сильный износ       |


**Зачем в проекте:**
Float влияет на цену скина. Один и тот же скин с Float 0.01 (почти идеальный) стоит в разы дороже, чем с Float 0.60 (потрепанный). Маркетплейс должен показывать Float и позволять фильтровать по нему.

---

### 13. Лутбокс / Кейс

**Простым языком:**
Кейс — это виртуальная "коробка с сюрпризом". Платишь деньги, открываешь, и получаешь случайный предмет. Может выпасть дорогой скин, а может дешевый. Это как автомат с игрушками в ТЦ — бросаешь монетку, получаешь случайную фигурку.

**Зачем в проекте:**
Кейсы — один из основных источников дохода сайта. Средний выигрыш всегда меньше стоимости открытия (иначе сайт был бы в убытке). Гарантированный приз = 50% от стоимости кейса.

---

### 14. Апгрейд

**Простым языком:**
Ты ставишь один или несколько дешевых скинов (и/или деньги с баланса) и пытаешься выиграть один дорогой. Чем дороже желаемый скин относительно твоей ставки, тем меньше шанс. Это как рулетка — поставил 1000 рублей, пытаешься выиграть скин за 10000 рублей с шансом ~8.5%.

**Формула:**
`Шанс = (Стоимость ставки / Стоимость приза) * (100% - 15% комиссия)`

Пример: ставка 1000р, желаемый приз 5000р → шанс = (1000/5000) * 85% = 17%

---

### 15. Анти-анлак система

**Простым языком:**
Если пользователь с Premium-подпиской открыл один и тот же кейс 10 раз подряд и ни разу не "окупился" (выигрыш каждый раз был меньше стоимости кейса), то 11-й раз — бесплатно. Это как "каждый 11-й кофе бесплатно" в кофейне, только привязано к неудачам.

**Техническая суть:**
Система хранит историю последних 10 открытий каждого пользователя. При открытии проверяет: все 10 одинаковый кейс + все 10 неокупаемые → следующий бесплатный.

### 16. Live-лента

**Простым языком:**
Бегущая строка в верхней части страницы кейсов, где в реальном времени появляются предметы, которые выиграли другие пользователи. Как новостная бегущая строка на телеканале, только с выигрышами.

**Режимы:**

- "ТОП" — только дорогие предметы (качество "Запрещенное" и выше)
- "ВСЕ" — все выигрыши

## Простые термины

---

### 17. API (Application Programming Interface)

**Простым языком:**
Это "меню ресторана" для программ. Как в ресторане ты говоришь официанту "принесите пиццу" (а не идешь на кухню сам), так и одна программа говорит другой через API "дай мне данные пользователя" или "создай трейд". API определяет, какие "блюда" (операции) доступны и как их "заказать" (формат запросов).

---

### 18. SPA (Single Page Application)

**Простым языком:**
Обычный сайт при переходе на другую страницу перезагружается целиком (мигание, белый экран). SPA — сайт, который загружается один раз, а дальше обновляет только нужные части без перезагрузки. Как приложение на телефоне — экраны переключаются плавно.

**В ТЗ это описано как:** "Навигация: Переключение вкладками (табами) без перезагрузки страницы" — это и есть SPA.

---

### 19. Пагинация

**Простым языком:**
Разбиение длинного списка на страницы. Как в Google — "Страница 1 из 100, следующая >". Вместо загрузки 10000 предметов разом (что было бы медленно), показываются по 20 штук на страницу.

**В ТЗ:** доступные значения — 10, 20, 50, 100, 500 предметов на страницу.

---

### 20. Локализация

**Простым языком:**
Перевод сайта на разные языки. Не просто текст — также формат дат, валют, чисел. Например, цена "1,000.50" в английском и "1 000,50" в русском.

**В ТЗ:** по умолчанию RU, EN, UZ. Новые языки добавляются через админ-панель.

---

### 21. Модальное окно

**Простым языком:**
Всплывающее окно поверх страницы. Как поп-ап "Вы уверены?" — затемняет фон, показывает содержимое, закрывается по крестику. Страница под ним не перезагружается.

**В ТЗ:** корзина, детали предмета, массовая продажа — все через модальные окна (не отдельные страницы).

---

### 22. Push-уведомления (браузерные)

**Простым языком:**
Уведомления, которые появляются на экране компьютера/телефона, даже если сайт закрыт. Как уведомления от приложений на телефоне, только от браузера. Сайт спрашивает "Разрешить уведомления?" — если да, может присылать сообщения.

---

### 23. SEO (Search Engine Optimization)

**Простым языком:**
Настройка сайта так, чтобы он появлялся выше в поиске Google/Яндекса. Включает: правильные заголовки страниц, описания, быструю загрузку, правильную структуру.

---

### 24. Open Graph

**Простым языком:**
Когда ты кидаешь ссылку в Telegram или VK, появляется красивая "карточка" с картинкой и описанием. Это работает благодаря Open Graph — специальным тегам на странице, которые говорят мессенджерам/соцсетям, как отображать ссылку.

---

### 25. bcrypt

**Простым языком:**
Способ "зашифровать" пароль так, чтобы даже если кто-то получил доступ к базе данных, он не смог узнать настоящий пароль. Это не шифрование (нельзя расшифровать обратно), а хеширование — одностороннее преобразование. При проверке пароля система хеширует введенный пароль и сравнивает с сохраненным хешем.

**В ТЗ:** код-пароль Premium хранится в bcrypt.

---

### 26. HTTPS

**Простым языком:**
Защищенная версия HTTP (протокола, по которому работают сайты). Все данные между браузером и сервером шифруются. Без HTTPS кто-то, подключенный к тому же Wi-Fi, мог бы перехватить пароли и данные. С HTTPS — не может. В браузере отображается замочек рядом с адресом.

---

### 27. Trade URL (трейд-ссылка)

**Простым языком:**
Специальная ссылка из Steam, которая позволяет другим людям (или сайтам) отправлять тебе предложения обмена без добавления в друзья. У каждого пользователя Steam своя уникальная ссылка.

---

### 28. Trade Ban (трейд-бан)

**Простым языком:**
Блокировка возможности обмениваться предметами в Steam. Может быть наложена Valve за подозрительные действия, или автоматически при смене пароля/устройства. Если у пользователя трейд-бан, он физически не может передать скин.

---

### 29. StatTrak

**Простым языком:**
Специальная версия скина с встроенным счетчиком убийств. На оружии отображается цифра — сколько врагов ты убил этим оружием. StatTrak-версии стоят дороже обычных.

---

### 30. Фаза (Phase)

**Простым языком:**
Некоторые скины (ножи и перчатки с покрытием Doppler) имеют разные цветовые вариации — фазы. Phase 1-4 — обычные, Ruby (красный), Sapphire (синий), Emerald (зеленый), Black Pearl — редкие и дорогие. Один и тот же скин в разных фазах может отличаться по цене в десятки раз.

---

### 31. Редкость (раритетность)

**Простым языком:**
Категория "крутости" скина. Определяет цвет подсветки карточки. От дешевых к дорогим:

- Ширпотреб (белый) → Промышленное (голубой) → Армейское (синий) → Запрещенное (фиолетовый) → Засекреченное (розовый) → Тайное (красный) → Контрабанда (золотой)

В ТЗ написано, что карточка предмета должна иметь "тень своей редкости" — цветное свечение вокруг карточки, соответствующее категории.

## Выводы

Основные "непонятные" моменты из ТЗ:

1. **"Приложение под браузер"** = браузерное расширение (Chrome Extension), маленькая программа, которая встраивается в Chrome/Firefox. Нужна только для продажи предметов.
2. **WebSocket** = постоянная связь с сервером для мгновенных обновлений (лента, уведомления).
3. **Атомарные операции** = защита от ситуации "деньги ушли, а предмет не пришел" — все финансовые операции выполняются целиком или не выполняются вообще.
4. **P2P** = напрямую между людьми, сайт — посредник только в деньгах, скины передаются напрямую через Steam.
5. **Нераспределенный фонд** = "копилка" кейсов, из которой система решает, можно ли выдать дорогой приз.
6. **Hold Balance** = замороженные деньги, которые ждут завершения сделки.
