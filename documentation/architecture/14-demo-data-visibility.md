# Демо-данные и флаг видимости

## Что считается демо

Данные, создаваемые `php artisan db:seed --class=DemoSeeder` (сидер не подключён к `DatabaseSeeder` по умолчанию). Маркеры заданы в `App\Support\DemoDataMarkers`:

- пользователи-продавцы: `steam_id` с префиксом `demo_` и `username` с префиксом `demo_seller_`;
- лоты маркета: продавец из этого набора или `asset_id` с префиксами `demo_asset_` / `demo_sold_`;
- кейсы: категория с именем «Демо-кейсы»; импорт из каталога (`CasesImportService`, категория «Официальные кейсы CS2») помечает кейсы как не ручные (`is_manual_admin_case = false`).

Удаление из БД: `php artisan demo:wipe` (или с `--force`).

## Флаг «Показ демо-данных»

Таблица `site_settings`, строка `id = 1`, поле `show_demo_data` (по умолчанию `true` после миграции). Чтение с кэшем — `App\Models\SiteSetting::showDemoData()`, запись с сбросом кэша — `SiteSetting::setShowDemoData(bool)`.

В админке Filament: раздел **Система** — страница **Демо-данные** (`ManageDemoVisibility`).

При **выключенном** флаге:

- публичный API скрывает демо в списках маркета, **кейсов** (остаются только `is_manual_admin_case = true` и не из категории «Демо-кейсы»), ленты открытий, сделок; отдельные `GET` по демолоту/скрытому кейсу — 404; покупка корзины с демолотом — ошибка валидации; открытие скрытого кейса — 404;
- SPA на главной не подставляет статичные картинки кейсов из `public/images/playstore`, если с API не пришли популярные кейсы;
- списки Filament (лоты, кейсы, пользователи, категории кейсов, уровни/предметы кейсов, балансы, транзакции) фильтруются теми же правилами. Новый кейс из админки получает `is_manual_admin_case = true` (`CreateGameCase`); импорт и демо-сидер — `false`.

Публичный эндпоинт для фронта: `GET /api/v1/site` — `{ "data": { "show_demo_data": true|false } }`, без авторизации.
